# 工具学习路径规划

> 目标：落地医院多类型报修工单系统 + AI 辅助课题论文写作

---

## 一、优先级总览

| 工具 | 与目标的关系 | 优先级 | 预计投入 |
|---|---|---|---|
| Supabase | 工单系统数据库+存储+权限，核心 | ★★★★★ | 2周 |
| GitHub | 网页托管 + 版本管理，必备基础 | ★★★★ | 1周 |
| NotebookLM | 课题文献综述、报告生成，立即可用 | ★★★★ | 半天 |
| Ollama | 本地大模型，处理含敏感信息的数据 | ★★★ | 2-3天 |
| Firebase | 与 Supabase 功能重叠，**跳过** | ★ | — |

> Firebase 和 Supabase 是同类竞品，已选 Supabase 后不需要再学 Firebase。

---

## 二、分阶段学习路径（4-6 周）

### 第 1 周：GitHub 基础
**目标**：把一个 HTML 文件发布成一个可访问的网址

- [ ] 注册 GitHub 账号
- [ ] 安装 GitHub Desktop（图形界面，避开命令行）
- [ ] 建仓库 → 上传 `index.html` → 开启 GitHub Pages
- [ ] 拿到 `https://你的名字.github.io/xxx/` 网址

**跳过**：分支、PR、Actions（当前用不到）

**验收**：把一张写着"病区报修"的网页发布出来，手机扫码能打开。

---

### 第 2-3 周：Supabase 入门（系统核心）
**目标**：建工单表，从网页提交数据，照片上传存储

- [ ] 注册 Supabase，建项目
- [ ] 建 `work_orders` 表（先只要 5 字段：病区、类型、描述、报修人、时间）
- [ ] Storage：传一张图片，拿到 URL
- [ ] **RLS（行级安全）**：必学，防止数据裸奔——Supabase 唯一难点
- [ ] Auth：暂跳过，第 4 周再加

**参考资料**：Supabase 官方文档 "Build a User Management App" 教程（照抄一遍）

**验收**：MVP 报修页能用，数据进库，照片能传 Storage。

---

### 第 4 周：NotebookLM（课题加速器）
**目标**：上传文献 PDF，生成综述草稿

- [ ] 上传 Clippings/ 里的文献（最多 50 篇/笔记本）
- [ ] 让它生成"研究空白"、"综述大纲"、"播客摘要"
- [ ] 跨文献找共识与分歧

**适合**：文献综述初稿、跨文献对比分析
**不适合**：含患者数据、写最终论文段落

**验收**：把"医院后勤数字化"主题 10 篇文献丢进去，生成 1500 字综述草稿。

---

### 第 5 周：Ollama（本地隐私 AI）
**目标**：在本机跑不联网的中文大模型，处理敏感数据

- [ ] 安装 Ollama（官网下载，一键安装）
- [ ] 运行 `ollama run qwen2.5:7b`（阿里中文模型，普通电脑可跑）
- [ ] 安装 Open WebUI 或 Cherry Studio（提供聊天界面）
- [ ] 测试：断网状态下分析一份带姓名的报修记录

**用途**：脱敏前的工单数据、含分机/姓名的内部表格、能耗数据分析

**验收**：全程断网，本地模型完成一份报修记录的故障类型分析。

---

### 第 6 周+：Supabase 进阶
- [ ] Realtime：新工单实时推送给组长
- [ ] Edge Functions：自动派单逻辑
- [ ] 与前端 JavaScript 完整集成

## 三、不学的内容

| 内容 | 原因 |
|---|---|
| Firebase | 与 Supabase 重叠，二选一即可 |
| GitHub Actions / 高级 Git | 当前阶段用不到 |
| Ollama 训练/微调 | 直接用现成模型已够用 |

---

## 四、注意事项

1. **合规先行**：开始 Supabase 前，先和信息科确认海外服务器是否合规
   - 若不合规，改用**腾讯云 CloudBase** 或 **Pocketbase 自托管**，架构完全相同
2. **MVP 优先**：第 1-3 周结束即上线最简版，不要等全套学完
   - 最简版 = 报修页 + 一张工单表 + 微信群人工派单
3. **学习笔记沉淀**：每完成一阶段，在 Obsidian 建对应学习笔记
