# 项目文件与图片索引规则

## 1. 项目聊天分支规则
同一个项目可以开多个独立聊天分支。
不同聊天分支可以共用项目文件，但不会自动共享完整上下文。
因此，重要结论、操作规则、目录结构和决策记录，应沉淀到项目文件中。

## 2. 自动化方向
后续可探索 computer use 自动化，用于执行重复性电脑操作。

## 3. Codex 历史同步
后续可研究 codex history sync tool，用于同步 Codex 操作记录、修改记录和项目历史。

## 4. 图片索引规则
图片资料只需建立一次统一索引，不需要每张图片在每个使用位置都重复写说明。
每周知识重组时，只对新增图片补充索引。

推荐文件：
- 图片索引.md
- 图片说明模板.md
- 图片引用记录.md


# CLAUDE.md｜医院后勤第二大脑设定

## 1. 关于我

- 我是医院后勤管理人员。
- 这个 vault 是我的医院后勤管理第二大脑。
- 主要管理内容：能耗管理、设备管理、维修巡检、工程验收、政策资料、制度流程、文献综述、汇报材料、论文和工作方案。
- 所有回应与笔记内容使用简体中文。
- 回答务实、准确、结构清晰，不过度发挥。

---

## 2. 全局铁律（最高优先级）

以下规则在整个 vault 中始终生效，后续章节不再重复：

1. **`Clippings/` 中的原始资料一律不修改、不移动、不删除**——只能读取、引用、摘要、建立索引。
2. **`创作库/` 中的成品不覆盖**——修改时新建版本文件（v1/v2/修改稿/定稿）。
3. **任何删除文件的操作必须先说明理由**，等待我确认。
4. **临时分析不得当作最终结论**——结论必须经过我确认才能写入创作库。
5. **敏感信息（供应商报价、招投标细节、人员投诉、患者相关信息）不得直接写入笔记**；必须引用时先脱敏（如"某供应商""某科室"）。

---

## 3. 笔记库结构

| 文件夹 | 角色 | 存放内容 |
|---|---|---|
| `Clippings/` | 输入层 | 网络文章、PDF 文献、政策原文、图片、截图、现场照片、原始资料 |
| `知识库/` | 消化层 | AI 整理后的结构化知识、索引、摘要、文献综述、主题分析 |
| `创作库/` | 输出层 | 论文、汇报材料、工作方案、制度文件、申报材料、公众号文章 |
| `每日笔记/` | 记录层 | 每日记录、周计划、知识重整周报 |
| `Templates/` | 模板 | 各类笔记模板 |

**层与维度的区别**：

- **文件夹按资料形态/角色组织**（输入/消化/输出/记录）。
- **主题（能耗、设备、工程验收等）通过 frontmatter 的 `tags` 表达**，不通过文件夹层层细分。

---

## 4. 文件归类规则

新增内容按性质放入对应位置：

- 别人的文章、政策原文、PDF、截图、现场照片 → `Clippings/`
- AI 整理后的摘要、分析、索引、知识卡片 → `知识库/`
- 我自己的论文、汇报、方案、制度、成品文章 → `创作库/`
- 每日记录、周计划、知识重整结果 → `每日笔记/`

**冲突归类原则**：

- 我对一份外部资料做了**批注或评论** → 资料原件留 Clippings，批注另起一篇放知识库，并 `[[链接]]` 回原件。
- 草稿尚未定稿 → 放 `创作库/草稿/` 子目录，定稿后移出。
- 不能确定时优先放 `知识库/`，并向我说明该资料的去向选择。

---

## 5. Frontmatter 规范（强制）

所有新建笔记必须包含以下 frontmatter：

\`\`\`yaml
---
title: ""              # 必填
date: YYYY-MM-DD       # 必填
type: ""               # 必填，取下方受控词表
tags: []               # 必填，至少一个主题标签
source: ""             # 引用外部资料时必填
status: ""             # 选填：draft / wip / final / archived
---
\`\`\`

### 5.1 type 受控词表

| 值 | 用途 |
|---|---|
| `clipping` | Clippings 中的剪藏笔记 |
| `note` | 知识库通用笔记 |
| `index` | 知识库索引页 |
| `summary` | 文献/政策摘要 |
| `review` | 文献综述 |
| `case` | 案例复盘 |
| `draft` | 创作库草稿 |
| `output` | 创作库定稿 |
| `daily` | 每日笔记 |
| `weekly` | 周计划/重整周报 |

### 5.2 tags 主题受控词表

主题标签从以下集合中选取，新增主题需先告知我：

\`\`\`text
能耗管理 / 设备管理 / 维修巡检 / 工程验收 / 政策研究 /
制度流程 / 文献综述 / 后勤管理 / 案例复盘
\`\`\`

辅助标签可自由添加（如 `汇报`、`待复查`、`季度任务`）。

## 6. 文件命名规范

### 6.1 普通笔记（md）

\`\`\`text
主题_对象_简要内容.md
\`\`\`

例：`能耗管理_住院楼_夜间基础负荷分析.md`

### 6.2 附件（图片、截图、现场照片、设备照片、PDF 等）

\`\`\`text
YYYY-MM-DD_主题_对象_简要内容_序号.扩展名
\`\`\`

例：

\`\`\`text
2026-05-03_能耗管理_住院楼_夜间电耗截图_01.png
2026-05-03_工程验收_门诊楼_空调管道保温_01.jpg
2026-05-03_设备管理_水泵房_控制柜报警_01.jpg
2026-05-03_政策研究_发改委_公立医院能效新规_01.pdf
\`\`\`

### 6.3 创作库版本命名

\`\`\`text
文件名_v1.md
文件名_v2.md
文件名_修改稿.md
文件名_定稿.md
\`\`\`

## 7. Clippings 目录结构(薄文件夹原则)

文件夹只按**资料形态 + 时间**做轻量分组,主题信息已在文件名和 tags 中体现,不再用文件夹层层细分:

\`\`\`text
Clippings/
├─ 网络文章/
├─ PDF文献/
├─ 政策资料/
└─ 图片资料/
   └─ YYYY-MM/         # 按年月归档
\`\`\`

不再设 `Clippings/图片资料/能耗截图/`、`Clippings/图片资料/设备照片/` 等主题子目录——主题通过文件名和索引检索。

---

## 8. 图片与附件规则

### 8.1 原件位置

图片、截图、现场照片、设备照片、PDF 等原件统一放在 `Clippings/` 下对应类别 + 年月目录,不复制、不移动到 `知识库/`。

### 8.2 命名时机(核心原则)

**重命名在"消化 Clippings"环节同步完成,不要拖到周度知识重整**。原因是 8.4 锁定了已引用的附件,安全窗口只在入库到首次引用之间。

- **入库即刻**:拍完照、剪藏完成时,如果当下就清楚是什么,直接按 6.2 命名好放入 Clippings——首选。
- **消化 Clippings 时**:对入库时未命名的文件,**在生成知识库笔记并首次引用之前**完成命名——主战场。
- **周度重整时**:兜底质检,复查是否有漏命名、漏索引、漏引用的情况——不是主战场。

### 8.3 允许重命名的附件

- 尚未建立索引、尚未被任何笔记引用、尚未用于创作库输出的新附件;
- 我明确要求整理命名的附件。

### 8.4 禁止直接重命名的附件

- 已被 Obsidian 笔记引用的;
- 已登记到索引的;
- 已用于创作库输出的;
- 无法判断来源、用途或关联关系的。

如确需重命名,必须**同步更新**:所有 Obsidian 引用路径、`知识库/00_索引/图片索引.md`、相关主题笔记、创作库引用文件。

### 8.5 创作库使用图片

- 优先使用 `[[Clippings/...]]` 链接引用原件。
- 对外提交材料时,可在项目目录下建 `assets/` 文件夹放副本:

\`\`\`text
创作库/项目输出/2026医院能耗分析汇报/assets/
\`\`\`

副本是副本,**不移动也不删除原件**。

## 9. 知识库维护

### 9.1 必备维护文件

\`\`\`text
知识库/index.md                # 知识库总入口与目录
知识库/log.md                  # 重整日志
知识库/00_索引/图片索引.md     # 图片附件索引
知识库/00_索引/PDF索引.md      # PDF 文献索引
\`\`\`

### 9.2 主题 MOC(可选)

按主题建 MOC(Map of Content)页面,作为该主题下笔记的入口:

\`\`\`text
知识库/MOC_能耗管理.md
知识库/MOC_设备管理.md
知识库/MOC_维修巡检.md
知识库/MOC_工程验收.md
知识库/MOC_政策研究.md
知识库/MOC_制度流程.md
知识库/MOC_文献综述.md
知识库/MOC_案例复盘.md
\`\`\`

MOC 通过 tag 查询动态聚合,**不需要把笔记搬来搬去**。

### 9.3 维护要求

- 新增知识库页面 → 更新 `index.md`。
- 知识重整 → 更新 `log.md`。
- 新增图片/PDF → 更新对应索引。
- 索引只记录元数据和链接,不保存副本。

## 10. 索引格式

### 10.1 图片索引

\`\`\`md
---
title: 图片索引
type: index
updated: YYYY-MM-DD
tags: [图片索引, 后勤管理]
---

# 图片索引

> 本文件只索引图片原件,不移动、不删除、不修改 `Clippings/` 中的原件。
> 表格中使用 `[[...]]` 链接引用,**避免使用 `![[...]]` 嵌入**——嵌入会试图渲染所有图片,导致索引页严重卡顿。

| 日期 | 原始文件名 | 标准文件名(链接) | 主题 | 内容说明 | 关联笔记 | 状态 |
|---|---|---|---|---|---|---|
| 2026-05-03 | IMG_0001.png | [[Clippings/图片资料/2026-05/2026-05-03_能耗管理_住院楼_夜间电耗截图_01.png|2026-05-03_能耗管理_住院楼_夜间电耗截图_01]] | 能耗管理 | 住院楼夜间电耗截图 | [[住院楼夜间基础负荷分析]] | 可用于汇报 |
\`\`\`

### 10.2 PDF 索引

格式同上。重要 PDF 建议同时在 `知识库/文献综述/` 下建一篇 `summary` 类笔记,索引中通过"关联笔记"列指向它。

## 11. 每日笔记与每周笔记

- `每日笔记/YYYY-MM-DD.md`:简短记录当日工作、想法、问题、待消化资料。
- `每日笔记/YYYY-Www_周计划.md`:周一列计划,周末检查完成度。
- `每日笔记/YYYY-Www_重整周报.md`:每周知识重整后的产出报告。
- 每日笔记中产生的有价值观点,在重整时**反向提炼**到知识库主题笔记,而不是把每日笔记直接搬过去。

---

## 12. 创作库规则

- `创作库/项目输出/项目名/` 是默认结构,每个对外项目独占一个目录。
- 项目目录下可有:`正文.md`、`assets/`、`参考资料.md`、`版本历史/`。
- 修改使用版本命名(见 6.3),不直接覆盖。
- 创作库内容可反向提炼到 `知识库/`,形成可复用观点、方法、框架、案例。

---

## 13. 每周知识重整流程

### 第一步:盘点本周变动

检查 `Clippings/`、`知识库/`、`创作库/`、`每日笔记/` 的本周新增和修改。

### 第二步:消化 Clippings(主战场,含命名)

- 阅读新增资料,提炼主题;
- **对涉及的图片、PDF 等附件,在生成知识库笔记并首次引用之前,完成标准化命名**;
- 生成或更新对应知识库笔记;
- 登记到 `知识库/00_索引/图片索引.md`、`知识库/00_索引/PDF索引.md`;
- 原件留在 Clippings,不移动。

### 第三步:附件命名兜底质检

复查本周是否还有:

- 漏命名的 `IMG_xxxx` 类原始文件名;
- 漏登记到索引的附件;
- 已被引用但命名仍不规范的附件(若有,按 8.4 同步更新所有引用)。

### 第四步:创作库回流

- 检查本周新增输出;
- 提炼可复用观点、框架、案例,更新到对应主题笔记。

### 第五步:知识库健康检查

- 孤立笔记(无入链);
- 应建未建的链接;
- 重复或冲突的主题页;
- 后续需要补充的资料缺口。

### 第六步:更新维护文件

\`\`\`text
知识库/index.md
知识库/log.md
知识库/00_索引/图片索引.md
知识库/00_索引/PDF索引.md
每日笔记/YYYY-Www_重整周报.md
\`\`\`

---

## 14. 常用指令

| 指令 | 含义 |
|---|---|
| 帮我新增到笔记 | 根据内容性质放入正确文件夹,补全 frontmatter |
| 消化 Clippings | 读 Clippings 新增资料 → 命名 → 生成知识库笔记 → 登记索引,原件不动 |
| 整理图片索引 | 扫描新增图片,必要时命名,更新图片索引;不移动、不删除、不改动已引用图片 |
| 整理 PDF 索引 | 同上,对 PDF 文献执行 |
| 跑一次知识重整 | 执行第 13 节完整流程 |
| 从知识库帮我写一份材料 | 优先引用知识库已消化内容,生成到创作库;不直接拼接 Clippings 原文 |

---

## 15. 禁止事项汇总

铁律已在第 2 节列出,以下为补充:

- 不把图片原件批量搬到 `知识库/`;
- 不在没有同步更新引用路径的情况下重命名已引用附件;
- 不在索引中使用 `![[...]]` 嵌入图片(用 `[[...]]` 链接);
- 不在 frontmatter 缺失的情况下提交笔记;
- 不直接把 Clippings 原文拼接为创作库成品(必须经过知识库消化);
- 不擅自新增 tags 受控词表外的主题词。

---

## 16. 最终执行原则

\`\`\`text
Clippings 保存原件
知识库 保存索引和消化后的知识
创作库 保存最终输出
每日笔记 记录过程,周度反向提炼到知识库
\`\`\`

附件治理:

\`\`\`text
入库即命名(首选)
消化时命名(主战场)
重整时质检(兜底)
引用后锁名(铁律)
\`\`\`
